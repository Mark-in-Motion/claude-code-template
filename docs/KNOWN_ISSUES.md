## How to use this file

- Add an entry when you discover a structural limitation you're choosing not to fix right now
- Include a workaround if one exists
- Update Status to `Fixed` and note the date when resolved
- Remove `Fixed` entries during quarterly cleanup to keep the list actionable

**Severity guide:**
- `High` — affects core functionality, workaround is painful
- `Med` — affects a non-critical path, workaround is acceptable
- `Low` — cosmetic or edge case, minimal impact
